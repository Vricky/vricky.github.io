hexo + github快速构建个人博客

2016-12-06

最近实在不知道要干什么，就想要不整个博客来写点东西，总不至于让自己的第一次实习生涯过的如此浑浑噩噩，其实记录东西用网上个各种博客网站都还好，但毕竟算是半个程序员，总要装逼性的提高一下自己的逼格，但要让自己从后台到前台写一个blog，还是觉得自己水平有限，所以就网上找了下比较方便有能装逼的方法，就看到hexo + github这个个人博客的构建，这里我就记录下自己在用hexo，github来建个人博客的过程，和其中自己遇到的问题。

（我是low的win系统）准备工作：

  1. github账号https://github.com/（是用来做博客的远程创库、域名、服务器之类的，怎么与本地hexo建立连接等下讲。）
  2. 安装node.js（hexo基于node.js环境）
  3. 安装git（为了把本地hexo上传到github上）

开始配置hexo：

找一个盘新建文件夹，可起名hexo，在其文件夹下右键打开git bash，在git bash里使用以下命令来安装hexo。

$ npm install hexo-cli -g

若npm速度太慢，我们可以使用阿里巴巴在国内的镜像服务器，命令如下：

npm install -gd express --registry=http://registry.npm.taobao.org

只需要使用–registry参数指定镜像服务器地址，为了避免每次安装都需要–registry参数，可以使用如下命令进行永久设置：

npm config set registry http://registry.npm.taobao.org

我用的是cnpm (gzip 压缩支持) 命令行工具代替默认的 npm:

$ npm install -g cnpm --registry=https://registry.npm.taobao.org

安装完成后可运行

hexo g
hexo d

来进行测试。

接下来就可以在hexo文件夹下来放置博客文件。点击鼠标右键，选择Git Bash，执行以下命令，Hexo会自动在该文件夹下下载搭建网站所需的所有文件。

$ hexo init

安装依赖包

$ npm install

让我们看看刚刚下载的hexo文件带来了什么，在E:\hexo内执行以下命令，

hexo g(部署)
hexo s（服务）

然后用浏览器访问http://localhost:4000/，此时，你应该看到了一个漂亮的博客了，当然这个博客只是在本地的，别人是看不到的，hexo3.0使用的默认主题是landscape。轻轻松松就看到了一点成果。

github建立需要使用的库：

大体方式就是：
new registry —> 填写用户名：*.github.io(这是因为github规定的格式这样才会直接讲当前仓库作为托管站点)，后续就可以不用管。

##部署本地文件到github上：##

既然Repository已经创建了，当然是先把博客放到Github上去看看效果。编辑hexo文件夹下的_config.yml文件（这是hexo的配置文件，在里面可以看到主页面上的显示）。

在_config.yml最下方，repository改成自己建的库。

deploy:
  type: git
  repository: git@github.com:Vricky/vricky.github.io.git
  branch: master

。另外记得一点，hexo的配置文件中任何’:’后面都是带一个空格的)。

我的_config.yml文件：

# Hexo Configuration
## Docs: https://hexo.io/docs/configuration.html
## Source: https://github.com/hexojs/hexo/

# Site
title: Blog·Vricky
subtitle: 多想过去的时间就是一场梦~
description: 多想过去的时间就是一场梦~
author: Vricky
language: zh-CN
timezone:

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://yoursite.com
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

# Directory
source_dir: source
public_dir: public
tag_dir: tags
archive_dir: archives
category_dir: categories
code_dir: downloads/code
i18n_dir: :lang
skip_render:

# Writing
new_post_name: :title.md # File name of new posts
default_layout: post
titlecase: false # Transform title into titlecase
external_link: true # Open external links in new tab
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
future: true
highlight:
  enable: true
  line_number: true
  auto_detect: false
  tab_replace:

# Category & Tag
default_category: uncategorized
category_map:
tag_map:

# Date / Time format
## Hexo uses Moment.js to parse and display date
## You can customize the date format as defined in
## http://momentjs.com/docs/#/displaying/format/
date_format: YYYY-MM-DD
time_format: HH:mm:ss

# Pagination
## Set per_page to 0 to disable pagination
per_page: 8
pagination_dir: page

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: hexo-theme-yilia

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: git@github.com:Vricky/vricky.github.io.git
  branch: master


avatar: /uploads/kobe.jpg
favicon: /uploads/favicon.ico

配置好_config.yml并保存后，执行以下命令部署到Github上。

hexo g
hexo d

如果你是第一次使用Github或者是已经使用过，但没有配置过SSH，则可能需要配置一下。
执行上面的第二个命令，可能会要你输入用户名和密码，皆为注册Github时的数据，输入密码是不显示任何东西的，输入完毕回车即可。

这样你就可以在网站上看到你刚部署上去的blog，https://vricky.github.io最主要的博客建立就完成了。

##另：发布新文章：##

1.在Git Bash执行命令：$ hexo new “my new post”（会在下述目录中建立文件）
2.在hexo文件夹下的source/_post中打开my-new-post.md，打开方式使用记事本或编辑软件。
hexo中写文章使用的是Markdown，没接触过的可以看下Markdown语法说明。

‘’—
title: 树莓派使用基本设置
tags:

  • 树莓派
  • 配置
    —‘’
    ##hexo一些常用命令：##
    hexo new”postName” #新建文章
    hexo new page”pageName” #新建页面
    hexo g #生成静态页面至public目录
    hexo s #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
    hexo d #将.deploy目录部署到GitHub
    hexo h # 查看帮助
    hexo version #查看Hexo的版本

使用hexo + github建博客就这么完成了，就是这么简单。因为本身自带的主题确实有点不怎么好看，下篇文章应该会记录更改主题的情况，其实也很简单，我更换的主题就是yilia，下篇文章会记录的。！！~~
